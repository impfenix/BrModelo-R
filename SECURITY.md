# Política de Segurança

## Reportando Vulnerabilidades
Se você encontrar uma vulnerabilidade de segurança, por favor, não abra uma issue pública. Em vez disso, envie um e-mail para `luana.dsl90@gmail.com`.

## Versões Suportadas
Apenas a versão mais recente do projeto recebe atualizações de segurança.

## Processo de Resposta
1. Recebimento do relatório.
2. Investigação e confirmação da vulnerabilidade.
3. Desenvolvimento de uma correção.
4. Lançamento de uma nova versão com a correção.
5. Divulgação pública (se necessário).
